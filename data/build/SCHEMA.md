# data/build/output/ — schema

Generated by `python3 data/build/build_fixtures.py`. Inputs: PPD:2026 (debt-earnings + institution-characteristics-and-completions) + IPEDS HD 2024 + IPEDS C 2024_A (Completions).

## Files

| File | Grain | Use |
|---|---|---|
| `output/institutions.json` | one row per UNITID | institution-picker UI; slim metadata |
| `output/by_unitid/<UNITID>.json` | one record per UNITID; programs nested | full slice consumed by rules engine |

Both files are regenerable from `data/build/raw/` and are git-ignored. Fixture for tests is committed at `tests/fixtures/sample-institution.json` (UNITID 110583, CSULB).

## `institutions.json` — directory record

```json
{
  "unitid": "110583",
  "opeid6": "001139",
  "instnm": "California State University-Long Beach",
  "stabbr": "CA",
  "control": 1,
  "iclevel": 1,
  "sector": 1,
  "n_programs": 194
}
```

| field | source | notes |
|---|---|---|
| `unitid` | IPEDS HD `UNITID` | string; primary key for the tool |
| `opeid6` | IPEDS HD `OPEID[:6]`, joined to PPD `opeid6` | string, zero-padded |
| `instnm` | IPEDS HD `INSTNM` | display name |
| `stabbr` | IPEDS HD `STABBR` | 2-letter state |
| `control` | IPEDS HD `CONTROL` | 1=public, 2=private not-for-profit, 3=private for-profit |
| `iclevel` | IPEDS HD `ICLEVEL` | 1=4-year+, 2=2-year, 3=<2-year |
| `sector` | IPEDS HD `SECTOR` | finer-grained ICLEVEL × CONTROL combination |
| `n_programs` | derived | count of program rows in the per-UNITID file |

### Program fields

| field | source | notes |
|---|---|---|
| `cip4` | PPD `cip4` | 4-character zero-padded string (e.g., `"5009"` for Music) |
| `cip4_title` | PPD `cip4_title` | trailing period preserved as published |
| `credlev` | PPD `credlev` | string label: `Bachelor`, `Master's`, `Doctoral`, `First Professional Degree`, `Associate`, `Undergrad Certificate`, `Post-Bacc Certificate`, `Graduate Certificate` |
| `cohort_count` | PPD `count_wne_p4` | integer or `null` if suppressed |
| `median_earn_p4` | PPD `md_earn_wne_p4` | float (USD) or `null` |
| `benchmark` | PPD `earn_bnchmrk_cip2_wageb` | float (USD) or `null`. Despite the `cip2` suffix in the PPD column name, this value is the OBBBA-aligned benchmark — see source-notes R08, R09 |
| `benchmark_route` | PPD `which_test_cip2_wageb` | the rule branch PPD selected: `Same-State HS Median`, `Same-State BA Median`, `Same-State, Same-Field BA Median`, `National HS Median`, `National BA Median`, `National Same-Field BA Median`, `Tie for Lowest Test`, `Not Listed in Section 84001` |
| `ep_gap_pct` | **derived** | `(median_earn_p4 - benchmark) / benchmark`, rounded to 4 places. NOT a published PPD field — see SPEC-DELTA |
| `ppd_fail_obbb` | PPD `fail_obbb_cip2_wageb` | 0 = pass, 1 = fail, `null` = not measured. R20 cross-validation reads this directly |
| `ppd_fail_master` | PPD `mstr_obbb_fail_cip2_wageb` | parallel Master's-only flag; `null` for non-Master's rows |
| `in_state_share` | PPD `pct_t4enrl_instate_p1819` ÷ 100 | float 0.0–1.0 or `null`. Institution-level value broadcast to every program record under the same OPEID6. R07 input. See "R07 in_state_share" below |
| `suppression.missing_test` | PPD `missing_test_cip2_wageb` | 1 = privacy-suppressed cell |
| `suppression.earn_suppressed` | derived | `true` if `median_earn_p4 is None` |
| `suppression.cohort_suppressed` | derived | `true` if `cohort_count is None` |
| `suppression.out_of_scope` | derived | `true` if `benchmark_route == "Not Listed in Section 84001"` |

The `suppression` block is a four-flag decomposition of what spec §3 calls a single "suppression flag". See SPEC-DELTA §3.

## R07 in_state_share

R07 in the rules engine asks "is this institution effectively in-state?" — UG benchmarking under OBBBA § 84001(c)(3)(B)(i) routes to the same-state HS median when in-state Title-IV-enrolled share is ≥50%, and to the national HS median otherwise.

**Source:** `pct_t4enrl_instate_p1819` from `ppd2026-institution-characteristics-and-completions.xlsx`. The variable expresses the share as a percentage (0–100); the build divides by 100 to store a fraction (0.0–1.0) on the JSON record.

**Grain:** institution-level. The PPD inst-completions file repeats the variable across every program row under a given `opeid6`; the build collapses on `opeid6` and broadcasts the value onto each program record so the rules engine reads the field locally without an institution-level join.

**SPEC-DELTA correction:** SPEC-DELTA §2.4 originally pointed at `financial-aid-part-1.xlsx` for this variable (citing the spec source-notes Pass-3 finding). Empirically the variable lives in the inst-characteristics file, not financial-aid-part-1. The bead-attached spec note allows for "successor variable name" precisely for this kind of reconciliation. fetch_ppd.py supports both `--only fa1` and `--only inst`; only the inst file is consumed by the build.

## R14 hidden_program_candidates

R14 surfaces programs that confer awards in IPEDS but are absent from PPD's per-cell program list — most often because the program does not receive Title-IV aid at the (cip4, credlev) cell, even though IPEDS Completions reports awards conferred under it. The B16 finding (Cinematic Arts BA at CSULB) is the prototype.

**Source:** IPEDS C2024_A (https://nces.ed.gov/ipeds/datacenter/data/C2024_A.zip), the Completions A subfile. Grain: UNITID × CIPCODE (6-digit, dotted) × MAJORNUM × AWLEVEL.

**Filters applied:**
- `MAJORNUM = '1'` — primary major only; avoids dual-major double-count.
- `AWLEVEL` mapped to PPD-style `credlev` via `AWLEVEL_TO_CREDLEV` in `build_fixtures.py`. Codes outside the map (e.g., the `99` totals row) are skipped.
- `CIPCODE` not starting `99` — drops totals/unknown placeholders.
- `CTOTALT > 0` — drops rows with no awards conferred.

**Detection:** for each UNITID, hidden = { (cip6, credlev) ∈ IPEDS } such that (cip4, credlev) ∉ { (cip4, credlev) of PPD program rows for this UNITID }, where `cip4 = first 4 digits of cip6 (dot stripped)`.

**Output (institution-level array):**

| field | source | notes |
|---|---|---|
| `cip6` | IPEDS `CIPCODE` | dotted format as published, e.g., `"50.0601"` |
| `credlev` | mapped from IPEDS `AWLEVEL` | uses the same string label set as PPD `credlev` (Bachelor, Master's, etc.) |
| `completers_total` | IPEDS `CTOTALT` (MAJORNUM=1) | integer; total completers across men + women |
| `vintage` | constant `"2024"` | the IPEDS C2024 release; awards conferred during AY 2023-24 |

The list is sorted by `(cip6, credlev)`. R14 in the rules engine reads this array; spec rule logic decides which candidates to elevate as advisory items in the verdict UI.

## Determinism guarantees

- All numeric fields are rounded at write time (gap to 4 places, money to 2 places).
- Program list within each institution sorted by `(cip4, credlev)`.
- Institutions list sorted by `(stabbr, instnm)`.
- Identical raw inputs → byte-identical JSON output.

```bash
python3 data/build/fetch_ppd.py --only debt
python3 data/build/fetch_ppd.py --only inst    # R07 in_state_share variable
python3 data/build/fetch_ipeds.py              # HD + C2024_A
python3 data/build/build_fixtures.py
```

A full refresh takes ~60s on a recent laptop (debt-earnings xlsx parse dominates).
